# Requirements: Trip Planner

**Defined:** 2026-07-14
**Core Value:** Provide a seamless, multi-interface collaborative trip planning experience that bridges Telegram group chats with web views.

## v1 Requirements

### Session Management

- [x] **SESS-01**: User session is saved globally at the application root and persists across browser page refreshes using browser `localStorage`.
- [x] **SESS-02**: Global Sign Out button is visible in the page headers to clear the session and return the user to the profile selector.

### Traveler Profiles

- [x] **PROF-01**: Display a visual, Netflix-like profile selector grid on `/trips` when no active user session is present.
- [x] **PROF-02**: Retrieve and display all existing traveler profiles dynamically from the `users` table, including search/filter filtering for scaling.
- [x] **PROF-03**: Allow registering a new traveler profile directly from the splash screen, generating a negative `telegram_id` to prevent collisions.

### Access Control & Privacy

- [x] **PRIV-01**: Restrict the `/trips` list dashboard to display only the trips where the active user is a participant.
- [x] **PRIV-02**: Query the database using Supabase joins (filtering the `rsvps` table by the active user's ID) to avoid downloading uninvited trip data client-side.
- [x] **PRIV-03**: Automatically register the creator of a new web trip as a `Committed` RSVP participant, making the trip immediately visible on their dashboard.

### Session & Roster Reconciliation

- [x] **RECON-01**: Reconcile the active user session with the trip's RSVP roster on `/trips/:id` details pages.
- [x] **RECON-02**: Block action interfaces and show a modal banner if a logged-in user visits a direct link of a trip they are not a participant of.
- [x] **RECON-03**: Provide a "Join Trip" button in the reconciliation banner to automatically insert an RSVP record for the active user.
- [x] **RECON-04**: Provide a "Switch Profile" option in the reconciliation banner to allow switching traveler profiles in-context.

### Telegram Deep Linking

- [x] **LINK-01**: Automatically authenticate and log in the user if query parameters (`tg_user_id` or `username`) are parsed from the landing URL.
- [x] **LINK-02**: Clean up the browser address bar immediately after processing query parameter authentication to prevent URL bookmark/sharing pollution.

## v2 Requirements

### Notifications & Integration

- **NOTF-01**: Automated settlement alerts pushed to Telegram from webapp webhooks.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Password-based authentication / OAuth | Deferred for v1. Simple passwordless traveler identity is sufficient and requested. |
| Role-based editing restriction | Any joined traveler can edit trip options, ledger, and RSVPs in v1.3. |
