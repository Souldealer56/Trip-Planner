# Requirements: Trip Planner (Telegram Bot Improvements)

**Defined:** 2026-07-12
**Core Value:** Provide a seamless, multi-interface collaborative trip planning experience that bridges Telegram group chats with web views.

## v1 Requirements

### Error Resilience (ERR)

- [x] **ERR-01**: The bot catches Supabase query, execution, and timeout exceptions globally.
- [x] **ERR-02**: The bot presents user-facing clean warnings when database operations fail, instructing them on how to retry.
- [x] **ERR-03**: The bot detects existing PM wizard states on new deep link start arguments and clears stale inputs gracefully.

### User Experience (UX)

- [x] **UX-01**: Uniform HTML text formatting styles applied across all bot command messages and rosters.
- [x] **UX-02**: Standardized custom emojis utilized consistently for rosters, options, expense logs, and confirmations.

### Pitching & Voting (PITCH)

- [x] **PITCH-01**: The PM option pitching wizard supports adding dynamic hyperlink URL fields.
- [x] **PITCH-02**: The PM option pitching wizard prompts for optional description details text.
- [x] **PITCH-03**: The bot displays a structured voting poll overview showing the tally of votes per pitched option.

### RSVP & Roster tracking (RSVP)

- [x] **RSVP-01**: The bot supports custom RSVP notes or additional participant details.
- [x] **RSVP-02**: Automatic background nudges notify group participants who have not RSVP'd yet.
- [x] **RSVP-03**: Automatic reminders alert participants to vote on pending polls before a stale threshold is reached.

## v2 Requirements

### Advanced Features

- **BUDG-01**: The web application supports custom expense splitting categories.
- **NOTF-01**: Web app triggers push notifications to Telegram users when new options are locked.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Direct Stripe Integration | Security risks and compliance overhead; settlement directions are sufficient for v1. |
| User Profile Avatars on Bot | Minimal visual utility inside group chat interface. |
