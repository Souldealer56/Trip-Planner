  <read_first>
    - web/verify-services.cjs
    - web/src/services/supabase.js
  </read_first>
  <action>
    Create a new file web/verify-options-service.cjs modeled after verify-services.cjs that:
    1. Parses the parent .env file to load SUPABASE_URL and SUPABASE_KEY.
    2. Initializes a Supabase client.
    3. Runs verification assertions for:
       - Creating a test option inside poll_options (trip_id: valid ID, category: 'accommodation')
       - Fetching options for the trip and category
       - Querying active_polls for the trip and category (creating a default row if it does not exist)
       - Casting a vote on the test option (adding option ID to voter_selections JSONB list for a mock user, updating votes_by_option tally)
       - Retracting the vote (removing option ID from voter_selections, updating votes_by_option tally)
       - Cleaning up the test option and resetting/deleting created active_polls rows
  </action>
  <acceptance_criteria>
    - web/verify-options-service.cjs exists.
    - Running `node web/verify-options-service.cjs` exits with code 0 on successful database integration checks.
  </acceptance_criteria>
</task>

<task type="auto">

  <read_first>
    - web/src/services/trips.js

    - web/verify-options-service.cjs

  <action>
    Create the options service file web/src/services/options.js and export:
    1. async function fetchOptions(tripId, category): Select * from poll_options matching trip_id and category, ordered by created_at ascending.
    2. async function pitchOption(tripId, category, name, estimatedCost, currency, link, description, addedByUserId): Insert a row with the provided option details.
    3. async function fetchActivePoll(tripId, category): Select active_polls row. If missing, insert a default row with empty voter_selections ({}) and votes_by_option ({}) and null telegram_poll_id.
    4. async function toggleVote(tripId, category, optionId, userId, cast): Retrieve/initialize the active_polls row, update voter_selections[userId] (adding/removing optionId as string), re-calculate votes_by_option tallies, save to active_polls, and return updated object.
  </action>
  <acceptance_criteria>
    - web/src/services/options.js contains exports: fetchOptions, pitchOption, fetchActivePoll, toggleVote.
    - Running `node web/verify-options-service.cjs` passes successfully.
  </acceptance_criteria>
</task>
