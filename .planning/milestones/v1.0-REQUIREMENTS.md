# Requirements

## v1 Requirements

### Setup & Configuration (SETUP)

- [x] **SETUP-01**: Initialize webapp project using React, Vite, and Vanilla CSS.
- [x] **SETUP-02**: Configure Supabase client connected to the existing project database.

### Trips View (TRIPS)

- [x] **TRIPS-01**: User can view a list of all trips from the database.
- [x] **TRIPS-02**: User can select a trip from the list to navigate to its detailed view.

### Trip Details (DETAIL)

- [x] **DETAIL-01**: User can view the title, destination, start/end dates, and base currency of a selected trip.
- [x] **DETAIL-02**: User can view a list of all participants RSVPed to the trip.

### Member Management (MEMBER)

- [x] **MEMBER-01**: User can select a participant from the list to "join the app" as that user, storing their session in `localStorage`.
- [x] **MEMBER-02**: User can add a new participant to the trip (adding to the database users/rsvps) and automatically join as them.

---

## v2 Requirements (Deferred)

- **EDIT-01**: Edit trip details (change destination, update dates) directly from the webapp.
- **PITCH-01**: Suggest/pitch new trip options (flights, hotels) through the webapp.
- **VOTE-01**: Cast votes on pitched options via webapp buttons.
- **EXPENSE-01**: Log expenses and see the trip ledger through the webapp.

---

## Out of Scope

- **AUTH-01**: Password-based signup or login — excluded for simplicity (replaced by shared-link access and participant selector).
- **REALTIME-01**: Web socket or real-time database listener syncing — deferred to prevent database query overhead; standard poll/refresh on navigation is sufficient.
