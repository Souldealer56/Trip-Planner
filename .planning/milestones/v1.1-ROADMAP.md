### 🚧 v1.1 Bot Capabilities & Improvements (In Progress)

**Milestone Goal:** Review and enhance the Telegram bot's capabilities, focusing on UX, voting, RSVPs, and error resilience.

#### Phase 6: Error Resilience & DB Hardening

**Goal**: Safeguard the bot's polling loop against connection dropouts or server failures when communicating with Supabase.
**Depends on**: Phase 5
**Requirements**: ERR-01, ERR-02, ERR-03
**Success Criteria** (what must be TRUE):

  1. The bot continues running even if Supabase goes offline or throws timeout errors.
  2. Users receive helpful text instructing them to retry when database updates fail.
  3. Re-entry into a PM wizard starts with a fresh session, clearing prior state inputs.

**Plans**: 1 plan

Plans:

- [x] 06-01: Global database client handlers and state cleanup re-entry routines (completed 2026-07-12)

#### Phase 7: UI & UX Enhancements

**Goal**: Make the bot's messaging formatting beautiful, consistent, and easy to read.
**Depends on**: Phase 6
**Requirements**: UX-01, UX-02
**Success Criteria** (what must be TRUE):

  1. All bot replies use uniform HTML tags and formatting.
  2. Roster snapshots, expense ledger tables, and option listings display with a consistent set of emojis.

**Plans**: 1 plan

Plans:

- [x] 07-01: HTML message templates, custom lists, and unified emoji indicators (completed 2026-07-12)

#### Phase 8: Option Pitching & Voting Upgrades

**Goal**: Upgrade option pitching and voting functionality to support URL hyperlinking, detail description fields, and structured vote tallies.
**Depends on**: Phase 7
**Requirements**: PITCH-01, PITCH-02, PITCH-03
**Success Criteria** (what must be TRUE):

  1. Users can add dynamic hyperlinks when pitching a trip option.
  2. Users can supply option description text during pitching.
  3. The bot prints a voting overview displaying the total vote counts per option.

**Plans**: 1 plan

Plans:

- [x] 08-01: Pitching descriptions, inline hyperlinks, and voting tally summaries (completed 2026-07-12)

#### Phase 9: RSVP Nudging & Roster Tracking

**Goal**: Implement automatic background reminders for RSVPs and unvoted polls.
**Depends on**: Phase 8
**Requirements**: RSVP-01, RSVP-02, RSVP-03
**Success Criteria** (what must be TRUE):

  1. The bot saves and displays participant notes or details on the roster.
  2. The bot automatically posts reminders to group chats for participants who haven't RSVP'd.
  3. The bot automatically nudges group chats for outstanding poll votes before deadlines.

**Plans**: 1 plan

Plans:

- [x] 09-01: Participant notes tracking, background reminders, and auto-nudges for RSVPs and voting (completed 2026-07-13)
