# Plan: Phase 6 - Error Resilience & DB Hardening

## User Review Required

> [!IMPORTANT]
> The database helper wraps synchronous Supabase queries using `asyncio.to_thread`. Since python-telegram-bot runs on asyncio, this prevents database timeout operations from blocking the bot loop for other active group users.

## Open Questions

None.

## Threat Model

<threat_model>
### Trust Boundaries
1. **Telegram Gateway Boundary:** Intercepts incoming client command payloads, deep links, and callback actions.
2. **Bot Process Boundary:** In-memory python-telegram-bot event loop running async handlers.
3. **Database Boundary:** External Supabase API calls.

### STRIDE Register
| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-01 | Denial of Service | Database Queries | High | MITIGATED | Wrapping all Supabase query execution calls in `asyncio.to_thread` to ensure worker thread pools absorb latency spikes and timeout stalls instead of freezing the async event loop. |
| T-06-02 | Elevation of Privilege | Deep Link Command Input | Medium | MITIGATED | Parse deep link command parameters using strict UUID regex validators to avoid processing injection payloads or malicious payload states. |
</threat_model>
